**Problem #11-1: Parallel File Processor**

**Objective**

Build a Parallel File Processor that uses QThreadPool to process multiple text files concurrently. Each file should be analyzed for word count, line count, and character count. Results are displayed in a Qt GUI with a progress indicator.

**Requirements**

**Part A: File Analysis Task**

- Create a FileAnalysisTask class that subclasses QRunnable.
- The run() method should open a text file, count lines, words, and characters.
- Use QMetaObject::invokeMethod to send the results back to the main thread.
- Handle file-open errors gracefully (emit an error signal/message instead of crashing).
- Set setAutoDelete(true) so the pool cleans up the task automatically.

**Part B: Qt GUI**

- Create a main window with: an "Add Files" button that opens QFileDialog::getOpenFileNames() to select multiple .txt files, a QTableWidget displaying results (columns: Filename, Lines, Words, Characters, Status), a QProgressBar that tracks how many files have been processed, a "Clear" button that resets the table and progress.
- When files are added, submit one FileAnalysisTask per file to QThreadPool::globalInstance().
- As each task completes, add a row to the table with the results.
- Show "Processing..." in the Status column while a file is being analyzed, then update to "Done" when complete, or "Error" if the file could not be opened.

**Part C: Thread Safety and Performance**

- Display the thread pool size (QThreadPool::globalInstance()->maxThreadCount()) in a QLabel.
- Add a QSpinBox to let the user change the thread pool size (range 1-16).
- Ensure all GUI updates happen on the main thread (no direct widget access from QRunnable).
- Add a "totals" row at the bottom of the table that sums all lines, words, and characters.
- Measure and display total processing time using QElapsedTimer.

**Problem #11-2: Real-Time Data Dashboard**

**Objective**

Build a Real-Time Data Dashboard that uses the Worker/moveToThread pattern to simulate live data arriving from sensors. The dashboard displays live statistics using a thread-safe data structure and updates the UI in real time using signals and slots.

**Requirements**

**Part A: Data Generator Worker**

- Create a DataGeneratorWorker class (QObject) that simulates sensor data.
- The worker should generate random temperature readings (between 15.0 and 35.0) every 200ms.
- Use moveToThread() to run the worker in a background thread.
- Emit a signal newReading(double value, QString timestamp) for each reading.
- Support start/stop via public slots. Use a boolean flag for cooperative cancellation.

**Part B: Thread-Safe Statistics Collector**

- Create a ThreadSafeStats class that uses std::atomic or std::mutex to safely track: total number of readings, running sum (for computing average), minimum and maximum values observed.
- Implement addReading(double value) that updates all statistics atomically.
- Implement getAverage(), getMin(), getMax(), getCount() methods that are safe to call from the GUI thread while the worker is adding readings.
- Implement reset() to clear all statistics.

**Part C: Qt Dashboard GUI**

- Create a dashboard with: four QLabel stat cards showing Count, Average, Min, and Max; a QListWidget showing the last 50 readings with timestamps; Start/Stop buttons to control the data generator; a Reset button to clear statistics.
- Use a QTimer (1-second interval) in the main thread to refresh the stat cards from the ThreadSafeStats object.
- Connect the worker's newReading signal to a slot that adds items to the QListWidget (limit to last 50 entries).
- When the user clicks Stop, the worker should stop generating data but the statistics should be preserved.
- When the user clicks Reset, clear both the stats and the list.
