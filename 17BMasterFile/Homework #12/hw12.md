# \*\*HW #12

(Network Programming Sockets & Client-Server Basics)\*\*

**Problem 12-1 Echo Server and Client:**

**Objective**

Build an Echo Server that receives messages from clients and sends them back (echoes them), plus a Qt GUI client that connects, sends messages, and displays the echoed responses. This is the "Hello World" of network programming.

**Requirements**

**Part A: Echo Server**

- Create a QTcpServer that listens on a configurable port.
- When a client connects, display a log message with the client's IP and port.
- When a client sends a message, echo it back with a prefix: "ECHO: " + original message.
- Support multiple simultaneous clients (store client sockets in a QList).
- When a client disconnects, remove it from the list and log the event.

**Part B: Client GUI**

- Create a main window with: QLineEdit for server address (default: "127.0.0.1"), QSpinBox for port (default: 12345), Connect/Disconnect buttons, QLineEdit for typing messages with a Send button, QTextEdit displaying all sent and received messages with timestamps.
- Use QTcpSocket to connect to the server.
- Display status (Connected/Disconnected) in a QLabel with color (green/red).
- Disable the Send button and message input when not connected.

**Part C: Message Framing**

- Use newline (\\n) as the message delimiter.
- Implement a buffer in BOTH the server and client to handle partial messages.
- Ensure messages are correctly reconstructed even if TCP splits them.

**Part D: Error Handling**

- Handle connection errors (server not running, wrong address) with a QMessageBox.
- Handle unexpected disconnection (server crashes) gracefully.
- Prevent sending empty messages.

**Problem 12-2 Multi-User Chat Application:**

**Objective**

Build a complete multi-user chat application based on the protocol taught in Lecture 12.3. The server broadcasts messages to all connected clients, and each client has a GUI showing the chat log and online user list.

**Requirements**

**Part A: Chat Server (15 points)**

- Implement the ChatServer class from Lecture 12.3 that supports JOIN, MSG, and QUIT commands.
- Maintain a QMap&lt;QTcpSocket\*, QString&gt; mapping sockets to usernames.
- Broadcast CHAT, JOINED, LEFT, and USERLIST messages according to the protocol.
- Create a simple server GUI window showing a log of all events (connections, messages, disconnections) with timestamps.
- Display the current number of connected users.

**Part B: Chat Client (20 points)**

- Implement the ChatClient class from Lecture 12.3.
- Create a client GUI with: a connection panel (host, port, username, Connect button), a QTextEdit showing the chat log with colored usernames, a QLineEdit + Send button for typing messages, a QListWidget showing online users.
- Parse all server messages (CHAT, JOINED, LEFT, USERLIST) and update the GUI accordingly.
- Show system messages ("Alice joined", "Bob left") in a different color (e.g., gray italic).
- Support pressing Enter in the message field to send.

**Part C: Message Framing and Robustness (10 points)**

- Implement proper message framing with newline delimiters and buffers on both server and client.
- Handle rapid message sending (multiple messages sent before readyRead fires).
- Handle client disconnection gracefully on both server and client sides.
- Prevent duplicate usernames (server rejects JOIN if username is already taken).

**Part D: Testing (5 points)**

- Test with at least 2 clients connected simultaneously to the same server.
- Include a screenshot showing multiple clients chatting.
- Include a screenshot of the server log showing connections and message routing.

Use the ChatServer and ChatClient classes from Lecture 12.3 as your starting point. The lecture code provides approximately 70% of the required functionality.
